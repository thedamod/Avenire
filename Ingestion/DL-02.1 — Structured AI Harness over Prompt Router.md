# Decision: DL-02.1 - Structured AI Harness over Simple Prompt Router

## Context
- Avenire requires orchestration for tutoring, tool calling, graph simulation, sandbox execution, memory, and reasoning transparency.
- A lightweight prompt-router approach risks entangled state and limited observability as complexity grows.
- The platform needs predictable reasoning behavior across evolving capabilities.

## Options Considered
1. Vercel AI SDK with light middleware
2. Simple ReAct-style prompting
3. Full harness with modular reasoning nodes and explicit phases

## Trade-offs
- Higher engineering complexity
- Slower initial implementation
- Stronger need for internal specification discipline
- Increased maintenance overhead

## Decision
- Build a structured AI harness with defined reasoning, execution, and reflection phases instead of a simple orchestrator.

## Rationale
- Explicit phase boundaries improve debugging, control, and extensibility.
- Tool execution and memory injection become modular rather than prompt-coupled.
- The harness supports pluggable reasoning strategies, controllable depth modes, and cost-governance mechanisms.

## Consequences
- Core infrastructure roadmap prioritizes observability and deterministic orchestration.
- Product iteration may start slower but supports long-term scale and reliability.
- Revisit trigger: if harness complexity materially slows iteration beyond acceptable strategic limits.
